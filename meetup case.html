Overview

As an immigrant and an avid traveler, I regularly use the Meetup app to find local events and likeminded people. I decided to redesign the Meetup app to learn more about UX/UI design principles and tools.

  • Product: Meetup app.
  • Target Audience: Immigrants, travelers, outgoing locals.
  • Challenge: Redesign the Home, Explore, and Notification pages.
  • Tools and Materials: Sketch; Balsamiq; Google; notebook; pencil; pen.

Some Meetup reviews on the Apple App Store.

User Research

In addition to my gripes with the app, I collected user data from Meetup reviewers on the Apple App Store.

Common User Complaints & Recommendations:

  • Display the distance to an event instead of (or in addition to) the venue name. Venue names are not very useful for non-locals or users that can’t read the local language.
  • On the Explore page, tapping the Add
    icon above an event adds users to the group, not the event. This is unintuitive and frustrating for users, especially because the action cannot be undone.
  • Add an Undo or Cancel button after each action so that the user feels in control.
  • On the Explore page, the top search bar only searches for events. The search bar for groups is located near the bottom of the page and is easily missed.
  • On the Home page, users cannot filter events from groups that they follow.

Sketches & Wireframes

To address user complaints without compromising aesthetics, I created low-fidelity sketches and wireframes of potential UIs.

  • Number of designs: 5
  • Key differences: Calendar vs. no calendar; horizontal vs. vertical scrolling; large vs. small images.

UI Mockups

After incorporating all of the user recommendations, I started designing the final screens in Sketch. Since most Meetup reviewers praised the app's UI, I decided to keep my designs as close to the original UI as possible to leverage the power of familiarity.

  • Guidelines: iOS Human Interface Guidelines; Existing Meetup color palette.
  • Platform: iOS

Home Page

Changes:

  • Replaced venue names with the distance to each event. Providing the distance to an event is useful for non-locals and people that can’t read the local language.
  • Added a Filter
    icon so users can filter events from groups that they follow.
  • Increased the font size of the event date and time so users can quickly orient themselves.
  • To free up white space, I removed See all groups and added an arrow next to Your Groups for the same functionality.
  • Placed group names below their respective pictures for better visibility and a higher contrast ratio.
  • Replaced Start group with a Create Group
    icon to encourage Meetup users to create groups. Groups are integral to the Meetup experience.
  • Added a thumbnail next to each event so users can quickly identify an event’s host group.
  • Changed Your Calendar to Your Meetups to incorporate branding and minimize confusion. A calendar is often used to track upcoming events, but people rarely track events that they aren’t attending.
  • Changed the Save icon from a star to a heart.

Potential Problems:

  • Changing Your Calendar to Your Meetups might not make sense to every user. User testing is required to make an informed decision.
  • Replacing the Save icon with a heart might not resonate with every user. Hearts are often associated with saving items, but this is not always the case. Icon functionality differs from site to site. Without user testing, it’s impossible to know whether the star or the heart is the better icon.

Explore Page

Changes:

  • Consolidated the event and group search bars into one Search bar located at the top of the page.
  • Added a Cancel
    icon so users can easily remove themselves from groups.
  • Separated groups and events so that the Add
    icon was unambiguous.
  • Increased the font size of upcoming dates (Ex. Today, Tomorrow).
  • Organized upcoming dates with a horizontal scroll feature.

Potential Problems:

  • Long group names might clutter the screen.
  • The Add and Cancel icons might be difficult to see depending on their background image.

Notifications Page

Many reviewers agreed that Meetup sends a lot of notifications, but the notification page itself is a bore to visit. I decided to change that.

  • Added an interactive questionnaire to catch the user’s attention.
  • Listed potential groups that the user might be interested in and added a Create Group
    icon.

Reflection

When I started this project, I had two goals in mind:

  1. Become proficient in Sketch and Balsamiq.
  2. Learn and apply the core principles of UI/UX design.

I've achieved the first goal and made progress on the second, but I have a long way to go until I can call myself a UI/UX designer.

In the field of user experience design, every decision must be backed by user data and perfected with prototype testing. Unfortunately, this project does not meet that mark. The reviewers on the Apple App Store might not represent Meetup’s target demographic, so there's a chance that my UI redesigns are ill-informed.
User tests are required to determine which interface–the redesign or the original–is preferred.

Next Steps:

  • Develop a working prototype in Figma.
  • Interview local Meetup users and record them using the original app vs. my prototype.
  • Analyze the data, address user pain points, and continue tweaking the UI in Sketch until it's perfect.
    (or until my 30-day free trial ends)
